# دلّة ☕ — دليل التشغيل والنشر (تطبيق ويب يُثبَّت على الجوال — PWA)

## هيكل المشروع
```
dalla-v3.html         ← الواجهة الكاملة
manifest.json          ← إعدادات التطبيق عند التثبيت (الاسم، الأيقونة، الألوان)
sw.js                  ← Service Worker (يُمكِّن التثبيت + العمل شبه-offline)
icons/                 ← أيقونات التطبيق (يمكنك استبدالها بتصميمك الخاص)
backend/                ← السيرفر المتصل بقاعدة بيانات PostgreSQL
```

---

## 📱 الآن تطبيقك يُثبَّت على الجوال فعلياً (PWA)

هذا تطبيق **PWA (Progressive Web App)** — نفس تجربة تطبيق حقيقي:
- أيقونة على الشاشة الرئيسية
- يفتح بدون شريط متصفح (ملء الشاشة)
- يعمل حتى لو الإنترنت ضعيف (الواجهة تُحمَّل من الذاكرة، البيانات الحقيقية من السيرفر)
- **بدون الحاجة لنشره على Google Play أو App Store**

### كيف يثبّته المستخدم؟
- **أندرويد (Chrome)**: يفتح الموقع → يظهر زر "تثبيت التطبيق على الجهاز" تلقائياً في شاشة الدخول → ضغطة واحدة
- **آيفون (Safari)**: يفتح الموقع → يضغط زر المشاركة ⬆️ → "إضافة إلى الشاشة الرئيسية" (التطبيق يوضّح له هذي الخطوات تلقائياً لو ضغط الزر)
- **كمبيوتر (Chrome/Edge)**: أيقونة تثبيت ⊕ تظهر في شريط العنوان

> لاحقاً إذا احتجت تطبيق حقيقي 100% على متاجر Google Play / App Store، نفس هذا الكود يصلح أساساً لتغليفه عبر أدوات مثل Capacitor أو TWA بدون إعادة بناء من الصفر.

### ⚠️ مهم: استبدل الأيقونات
الأيقونات الموجودة في `icons/` مؤقتة (تصميم بسيط بفنجان قهوة). استبدلها بشعارك الفعلي بنفس الأسماء والأحجام:
- `icon-192.png` (192×192)
- `icon-512.png` (512×512)
- `icon-192-maskable.png` (192×192، بهامش أمان للأيقونات الدائرية على أندرويد)
- `icon-512-maskable.png` (512×512، نفس الفكرة)

## 🚀 النشر على Render

### الخطوة 1: ارفع المشروع على GitHub
```bash
git init
git add .
git commit -m "دلّة - تطبيق PWA + قاعدة بيانات"
git remote add origin <رابط مستودعك>
git push -u origin main
```

### الخطوة 2: أنشئ Blueprint على Render
1. سجّل دخول إلى [render.com](https://render.com)
2. **New** → **Blueprint** → اربط المستودع
3. سيكتشف `backend/render.yaml` وينشئ السيرفر + قاعدة بيانات PostgreSQL تلقائياً
4. اضغط **Apply** وانتظر اكتمال النشر (~3-5 دقائق)

### الخطوة 3: أنشئ الجداول
من تبويب **Shell** في خدمة `dalla-backend`:
```bash
npm run db:init
```

### الخطوة 4: أنشئ أول حساب أدمن
```
POST https://dalla-backend-xxxx.onrender.com/api/auth/bootstrap-admin
Content-Type: application/json

{ "phone": "admin001", "name": "مدير المنصة", "pin": "0000" }
```
(يعمل مرة واحدة فقط، يُغلق تلقائياً بعدها)

### الخطوة 5: اربط الواجهة بالسيرفر
افتح `dalla-v3.html` وعدّل قرب البداية:
```js
const PRODUCTION_API_URL = "https://dalla-backend-xxxx.onrender.com";
```

## 🖥️ التشغيل محلياً

```bash
cd backend
mv env.example.txt .env     # عدّل DATABASE_URL بداخله
npm install
npm run db:init
npm start
```

السيرفر يخدم تلقائياً: الواجهة + الـ API + manifest.json + sw.js + الأيقونات، كلها من نفس الرابط `http://localhost:4000` — افتحه في المتصفح وجرّب زر "تثبيت التطبيق".

---

## 🔑 بيانات الدخول
لا توجد بيانات تجريبية. أنشئ حساب الأدمن أولاً (الخطوة 4)، ثم:
- **الزبون**: تسجيل دخول برقم الهاتف عبر OTP (يظهر في Logs السيرفر)
- **الباريستا/المدير**: ينشئهم الأدمن من لوحته، أو عبر `POST /api/admin/staff`

---

## 📌 ملاحظات
- **بدون MongoDB إطلاقاً** — قاعدة البيانات الوحيدة هي PostgreSQL.
- **بدون بيانات تجريبية** — كل شيء يُضاف من لوحة الأدمن.
- **خطة Render المجانية لقاعدة البيانات تُحذف بعد 90 يوماً** من الإنشاء — رقّها لخطة مدفوعة إذا أردت بقاء دائم للبيانات.
- رمز OTP يُطبع حالياً في سجلات السيرفر فقط (للتجربة). لإرسال SMS حقيقي، أضِف مزوّداً (Twilio/Unifonic) في `backend/routes/auth.js`.

---

## 🔄 إصلاح: لماذا لا تظهر الكوفيهات/المنيو الجديدة عند الزبون؟

**السبب القديم:** كانت إضافة كوفي أو مشروب من لوحة الأدمن تُحفظ في **متغير JavaScript محلي بالمتصفح فقط**، ولا تُرسَل لقاعدة البيانات إطلاقاً. لذلك لا تظهر عند أي مستخدم آخر، وتُفقد بمجرد إغلاق التطبيق.

**تم الإصلاح بالكامل:**
- إضافة/تعديل كوفي → يُرسَل فعلياً إلى `POST/PATCH /api/cafes`
- إضافة/حذف مشروب → يُرسَل فعلياً إلى `POST/DELETE /api/cafes/:id/menu`
- زر التحديث العلوي وسحب الشاشة للتحديث → يُعيدان **جلب** البيانات من السيرفر فعلياً (وليس مجرد عرض رسالة)
- مزامنة تلقائية كل 20 ثانية بين كل التطبيقات (تجلب فعلياً، لا تعيد بث بيانات قديمة)

> **مهم:** هذا الإصلاح يعمل فقط إذا كان `PRODUCTION_API_URL` مضبوطاً بشكل صحيح، والسيرفر متصل بقاعدة البيانات فعلاً (تحقق من شارة "متصل بقاعدة البيانات" في شاشة الدخول).
